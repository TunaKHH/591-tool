# 設計:「已看過」物件標記

日期:2026-06-25 · 專案:591 租屋列表過濾(Chrome 擴充,Manifest v3,v1.5)

## 背景:為什麼做這個

從 Chrome Web Store 後台數據回推(整個生命週期約 272 安裝 / 15 解除安裝,留存
~94.5%),**流失不是問題,成長才是**。而成長的卡點在於產品價值沒講清楚——目前對外
是「隱藏 591 列表物件」,功能導向、弱。

重新釐清核心用戶問題:

> 找租屋是橫跨數天到數週、用同一組篩選反覆刷新的過程,但 591 列表是**無狀態的**——
> 它不記得你看過、淘汰過、聯絡過哪些。每次回來都得在重複物件裡重新撈新的,又累又易漏。

本功能聚焦其中最有感的一刀:**「我點進去看過詳情的物件,回到列表分不出來,常常重複
點開同一間。」**(等同瀏覽器「已造訪連結」變色,但 591 做得爛、重整就沒了。)

這同時是新的行銷鉤子,供日後 Store 文案 / 社群貼文使用。

## 核心行為

- 在列表頁點任一物件進詳情(含中鍵 / Ctrl+ 點開新分頁)→ 該物件 ID 標記為 `seen`
  + 時間戳。**只有「點進詳情」算看過**(明確意圖訊號,不用 dwell / IntersectionObserver)。
- 回到列表 / 重整 / 換篩選時,逐卡判斷:
  - `removed`(手動淘汰)→ **完全隱藏**(現有功能,不動)
  - `seen`(點過的)→ 依設定 **淡化**(預設:opacity + 灰階)或 **完全隱藏**

> 註:trigger B 之下不做「新物件 badge」——沒點過的卡片佔絕大多數,全標「新」會變雜訊。

## 元件改動

1. **`content-script.js`**
   - 攔截卡片連結點擊(capture 階段 + `auxclick` 接中鍵)→ 抓物件 ID → 寫入 `seen`
   - 套用視覺處理:頁面載入時 + `MutationObserver`(接 591 SPA 局部更新)對 `seen`
     卡片加 class(淡化或隱藏)
   - `storage.onChanged` 監聽 → 設定一改即時生效,不需重整
2. **儲存 schema**(`chrome.storage.local`):在現有 `removed` 旁新增
   `seen: { 物件ID: 時間戳 }`
3. **`options.html` / `options.js`**
   - **設定區**(頂部):開關「已看過物件 → 淡化 / 隱藏」(預設淡化)
   - **分頁切換**,沿用現有可排序表格元件:
     - 已移除(現有)
     - 已看過(新):欄位沿用(物件名稱 / 時間 / 排序);操作:**還原單筆**、
       **清空全部**(顯眼,最常用)
   - 匯出 / 匯入:**納入 `seen`**(沿用現有「已移除」同一套機制)

## 資料流

```
點擊物件 → seen[id] = now → storage.local.set
列表渲染 → 讀 removed + seen + 設定 → 逐卡:removed→hide;seen→(設定?hide:fade)
設定切換 → storage.local.set → content script storage.onChanged → 即時重套
```

## 錯誤處理 / 邊界

- 抓物件 ID 失敗 → 該卡跳過,**絕不弄壞列表渲染**
- `seen` 會成長(每次點擊都記)→ 以時間戳**自動清理超過 ~90 天的**項目(物件本來就
  會下架);配合「清空全部」避免無限膨脹
- 591 SPA 局部更新 → `MutationObserver` 接住新卡片再套規則(沿用現有「移除按鈕」
  注入機制)
- 全程**純本地、不上傳、不需新權限**(維持只有 `storage`)——本身也是賣點

## 測試(手動,在真實 591 列表頁)

1. 點一個物件進詳情 → 返回 → 確認依設定淡化 / 隱藏
2. 重整頁面 → 標記仍在
3. 設定頁切換「淡化 ↔ 隱藏」→ 列表即時變化(免重整)
4. 中鍵 / Ctrl+ 點開新分頁 → 也算看過
5. 換篩選條件(SPA 更新)→ 規則正確重套
6. 匯出 → 清空 → 匯入 → `seen` 與 `removed` 都復原

## 範圍外(先不做)

- **重複刊登去重**(同一房、代管用不同 post id 洗版):需「地址 + 價格 + 坪數」模糊
  比對,屬另一個功能,不混入本次。
- 自動淡化(滑進視窗 + 停留):已評估後否決,改採 trigger B(點進詳情)。

## 後續(本功能完成後)

用新鉤子翻修成長動作:Store 標題 / 第一張截圖 / 描述前兩行打「點過的物件回到列表自
動標記,不再重複點開」;社群貼文(Threads / Dcard 租屋 / FB 租屋社團)以同一痛點切入。
